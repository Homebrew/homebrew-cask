# Fix: `name` undefined in preflight heredoc in `trae-cn.rb`

## Root Cause

In the `on_linux > preflight` block (lines 52–98), the heredoc uses `#{name}` and `#{desc}` for string interpolation. The error "name is unknown method" occurs because inside the `preflight do...end` block, the Ruby `self` context has changed — it is no longer the cask DSL instance. The `name` and `desc` DSL methods are not available in that scope.

This is standard Ruby behavior: DSL methods like `name` and `desc` are instance methods on the cask object. Inside blocks that are evaluated with a different receiver (via `instance_eval` or similar), those methods are no longer on `self`.

## Fix

Capture `name` and `desc` as local variables **before** the `preflight` block where the cask DSL context is still active, then use those variables inside the heredoc interpolation.

### File to modify
- `/var/home/min/repos/github/homebrew-cask/Casks/t/trae-cn.rb`

### Changes

1. **Add local variable captures** inside the `on_linux do` block but before `preflight do`:

```ruby
on_linux do
  artifact_arch = on_arch_conditional arm: "aarch64", intel: "x86_64"
  cask_name = name    # capture DSL value as local variable
  cask_desc = desc    # capture DSL value as local variable
```

2. **Replace `#{name}` with `#{cask_name}`** in both heredocs (lines 56, 86)
3. **Replace `#{desc}` with `#{cask_desc}`** in both heredocs (lines 57, 87)
